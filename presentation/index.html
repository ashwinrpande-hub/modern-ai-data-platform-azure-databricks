Ingestion estate

Three templates land everything in Bronze

Any technical person onboards a source with one YAML row + one command. No new code, ever.

Oracle / EBS ─ GoldenGate Extract ─► OGG BigData handler ─► ADLS (avro_op: op_type + CSN)
            ─► Auto Loader ─► DLT APPLY CHANGES INTO bronze.sap.vbak  -- keys=PK, sequence_by=csn

Reuses existing OGG investment; landing zone decouples capture from compute. Swap-ready: when Lakeflow Connect Oracle is GA for our tenant, the YAML pattern: changes — nothing else.

SQL Server (CDC / Change Tracking) ─► Lakeflow Connect gateway (in our VNet)
            ─► staging volume ─► managed pipeline ─► bronze.qad.so_mstr

Replaces Fabric Mirroring: one governance plane (Unity Catalog), no OneLake copy, serverless processing. Source-side prep is a two-line script any DBA runs.

PLCs / SCADA ─► Litmus Edge (UNS: nucor/<site>/<area>/<asset>/<tag>)
   ├─ default:      ADLS parquet ─► Auto Loader ─► bronze.ot.*
   └─ low-latency:  Event Hubs (Kafka API) ─► Structured Streaming ─► bronze.ot.*  -- watermark 10m

Litmus keeps doing what it's good at — edge normalization. Topic→table mapping is config; the template enforces one OT envelope: site, area, asset, tag, ts, value, quality.

The shared contract: same envelope columns, same DQ gate, same quarantine table, same registration via deploy.py --register-source.

Medallion architecture

Data rolls through the mill — click a stand

Bronze — schema-on-write mirror, schema-on-read escape hatch

Source 3NF structures preserved; every table carries _source_system, _ingest_ts, _batch_id, _op_type, _seq, _raw. DLT expectations quarantine bad rows into audit.rejected_records with the failed rule and DAMA dimension.

Silver design

Insert-only 3NF with deterministic hash keys

hk = sha2(concat_ws('|', source_system, original_pk), 256)

-- SAP order 0004000123  → hk a91f…  (always)
-- JDE order 00001-100042 → hk 7c2e…  (always)

-- no UPDATE / DELETE ever issued:
new version  ⇒  new row (effective_ts)
"current"    ⇒  view: row_number() over (partition by hk
                       order by effective_ts desc) = 1

Why this wins

  • Deterministic across reloads & environments — same input, same key, no lookup tables
  • Streaming-friendly — append-only writes, no MERGE contention
  • Audit for free — full history is just the table
  • Gold assembles by key — facts join dims on hk, no surrogate pipeline
  • record_hash detects no-op updates; quarantine + batch_log on every run

The loop, not just the score

  • DLT expectations gate in-pipeline; framework scores cross-table per dimension
  • Row-level rejects → audit.rejected_records (rule + dimension + payload)
  • Threshold breach fails the job → webhook → DQ RCA agent finds first bad batch & blast radius
  • Healer agent: schema drift → mapping PR; infra → smart retry; never auto-merges
  • Scores published per data product = its trust score
